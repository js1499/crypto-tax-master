# US Securities Tax Engine — Implementation Plan

## Architecture Decision

The securities engine is a standalone system sharing infrastructure (database, auth, UI framework) but completely independent from the crypto engine. Separate data models, separate calculation pipelines, separate report generation. The two engines never share lots, wash sale detection, or tax events. Each produces its own complete set of IRS forms.

The database schema uses a `securities_` prefix for all new tables to avoid any collision with the crypto transaction tables.

### Step 2: CSV Import System

**File: `src/lib/securities-csv-parser.ts`**

Parse the universal CSV template defined in the requirements doc. Validate all fields per Section 2.5 validation rules. Return structured transaction objects ready for database insertion.

Key validations:
- Required fields present per transaction type
- Quantity/price/fees are valid numbers
- Date is valid and not in the future
- Option symbols follow OCC format
- Asset class is valid enum value
- Duplicate detection (same date/type/symbol/qty/price)
- Account type is valid
- Total amount consistency check vs price × quantity ± fees

**File: `src/app/api/securities/import/route.ts`**

POST endpoint accepting CSV upload. Parses, validates, inserts into `securities_transactions`. Returns import summary with warnings/errors.

### Step 3: Lot Tracking Engine

**File: `src/lib/securities-lot-engine.ts`**

Core engine that processes transactions chronologically and maintains lot state.

Functions:
- `processSecuritiesTransactions(userId, year)` — main entry point
- `createLot(transaction)` — creates a new lot from a buy/acquisition
- `consumeLots(symbol, quantity, method, account)` — selects and consumes lots per cost basis method
- `applyStockSplit(symbol, ratioFrom, ratioTo)` — adjust all open lots
- `applyDividend(transaction)` — record dividend, create DRIP lot if reinvestment
- `applyReturnOfCapital(symbol, amount)` — reduce basis across lots
- `generateDeemedSales(year)` — for Section 475 MTM and Section 1256 year-end
- `generateSection481Adjustment(year)` — transition year deemed sales

Lot selection methods:
- FIFO: sort by date_acquired ASC
- LIFO: sort by date_acquired DESC
- HIFO: sort by cost_basis_per_share DESC
- Specific ID: match by lot_id from CSV
- Average Cost: (total cost / total quantity) for mutual funds only
- Minimum Tax: long-term gains first, then long-term losses, then short-term gains, then short-term losses

### Step 4: Wash Sale Engine

**File: `src/lib/securities-wash-sale-engine.ts`**

The most complex component. Iterative multi-pass engine.

```
function detectWashSales(userId, year, taxSettings):
  1. Collect all loss sales for the year (+ December of prior year for cross-year)
  2. For each loss sale:
     a. Find substantially identical acquisitions in 61-day window (30 before, 30 after)
     b. Check 30-day lookback (before sale) first — takes priority
     c. Match acquisitions FIFO within the window
     d. Compute disallowed amount (prorated if partial)
     e. Adjust replacement lot: add disallowed loss to basis, tack holding period
     f. If replacement is in IRA/retirement: mark as permanently disallowed
     g. Record wash sale in securities_wash_sales table
  3. Check for daisy-chains: did any basis adjustment create a new loss?
  4. Repeat from step 2 until no new wash sales (max 100 iterations)
  5. Generate carry-forward records for December losses deferred into next year
```

Substantially identical matching:
- Same symbol: automatic
- Options on same underlying: configurable (Method 1 = all options, Method 2 = same strike/type/exp)
- User-defined equivalence groups: check `securities_equivalence_groups`

Cross-account scanning:
- Scan ALL user accounts (taxable + retirement)
- Taxable → taxable replacement: loss deferred, basis adjusted
- Taxable → IRA/retirement replacement: loss permanently disallowed

### Step 5: Section 1256 Engine

**File: `src/lib/securities-section-1256.ts`**

- Identify qualifying contracts by symbol lookup against `securities_section_1256_symbols`
- Year-end deemed sales for open 1256 positions (using YEAR_END_FMV transactions)
- 60/40 split: each gain/loss event generates two sub-events (60% long-term, 40% short-term)
- Route to Form 6781

### Step 6: Section 475 MTM Engine

**File: `src/lib/securities-section-475.ts`**

- For TRADER_MTM users: year-end deemed sale of ALL open positions at FMV
- All gains/losses become ordinary (not capital)
- Bypass wash sale engine entirely
- Generate Section 481(a) adjustment for transition year
- Route to Form 4797

### Step 7: Section 988 Forex Engine

**File: `src/lib/securities-section-988.ts`**

- Forex gains/losses are ordinary income by default
- If user elected out (section_988_election = true): treat as capital (and Section 1256 if qualifying)
- Currency futures on regulated exchanges: always Section 1256 regardless of election
- Route to Schedule 1 Line 8z (investors) or Schedule C (traders)

### Step 8: Dividend and Income Engine

**File: `src/lib/securities-dividends.ts`**

- Process DIVIDEND, DIVIDEND_REINVEST, INTEREST, RETURN_OF_CAPITAL transactions
- Classify by dividend_type from CSV
- DRIP: create new lot with acquisition date = payment date, cost = reinvestment price
- Return of capital: reduce basis proportionally across all lots for the symbol
- Track totals for Schedule B generation
- Foreign tax paid tracking for Form 1116 reference

## Phase 2 — Corporate Actions, Options, Employee Equity

### Step 12: Corporate Actions Engine

**File: `src/lib/securities-corporate-actions.ts`**

- Stock splits: adjust all open lots (quantity × ratio, cost ÷ ratio)
- Reverse splits: adjust lots, generate cash-in-lieu disposal for fractional shares
- Mergers (stock-for-stock): re-symbol lots, adjust quantity by exchange ratio, preserve basis/dates
- Mergers (cash): treat as sell at merger price
- Mergers (mixed): cash portion triggers gain, stock portion carries over
- Spinoffs: allocate parent basis by allocation_pct, create new lots for spinoff with inherited dates
- Ticker changes: rename across all open lots
- Return of capital: reduce basis, excess becomes capital gain

### Step 13: Options Engine

**File: `src/lib/securities-options.ts`**

- Long call exercise: premium + strike = stock cost basis, option lot consumed
- Short call assignment (covered call): shares sold at strike, premium added to proceeds
- Long put exercise: shares sold at strike, premium reduces proceeds
- Short put assignment: shares acquired at strike, premium reduces cost basis
- Expiration: long = capital loss (ST or LT), short = short-term gain (always)
- Close (buy/sell to close): standard capital gain/loss on option contract
- Section 1256 index options: auto-identify, 60/40 treatment, Form 6781

### Step 14: Short Sale Engine

**File: `src/lib/securities-short-sales.ts`**

- SELL_SHORT: record proceeds, no taxable event, create "short lot"
- BUY_TO_COVER: close short, gain/loss = short proceeds - cover cost - fees
- Holding period: generally short-term
- Year-end reporting: use closing date for reporting purposes
- Wash sales: cover at loss + re-short within 30 days = wash sale
- Constructive sale flagging (Section 1259): detect long + short on same security

### Step 15: Employee Equity

**File: `src/lib/securities-employee-equity.ts`**

- RSU vest: create lot at FMV, handle sell-to-cover (immediate zero-gain disposal of withheld shares)
- ESPP purchase: create lot at discounted price, track offering_date for qualifying/disqualifying disposition determination
- ESPP disposition: compute ordinary income portion vs capital gain based on holding period relative to offering date and purchase date

## Phase 3 — Differentiation

### Step 19: Tax-Loss Harvesting Report

- Scan all open lots for unrealized losses
- For each: check if harvesting would trigger a wash sale (30-day purchase history)
- Compute estimated tax savings at user-specified marginal rate
- Flag short-term losses (higher tax value)
- Suggest non-substantially-identical alternatives (requires security classification database)

### Step 20: Market Data Integration

- Integrate market data API (Alpha Vantage, Polygon, or Twelve Data) for current prices
- Auto-populate year-end FMV for Section 1256 and Section 475 positions
- Real-time unrealized gain/loss calculations

### Step 21: Section 1092 Straddle Engine

- Auto-detect offsetting positions (long stock + long put, etc.)
- Present flagged straddles for user confirmation
- Loss deferral: defer closed-leg losses to extent of open-leg unrealized gains
- Holding period suspension while straddle exists
- Identified straddle election workflow
- Form 6781 Part II reporting

### Step 22: Minimum Tax Cost Basis Method

Automated specific identification that minimizes current-year tax:
1. Sell long-term gain lots first (lowest rate)
2. Then long-term loss lots
3. Then short-term gain lots
4. Then short-term loss lots (highest tax value — save for last)

### Step 23: Section 1256 Loss Carryback

Generate data for 3-year carryback election:
- Prior 3 years' Section 1256 gains
- Carryback allocation worksheet
- User files amended returns manually

### Step 24: Worthless Securities

Transaction type `WORTHLESS`: deemed sale on December 31 at $0. Capital loss = full cost basis. Common for bankruptcies, delisted stocks, failed SPACs.

### Step 25: Multi-Year Data Reconstruction

Import multiple years of history and compute running lot state across years. Carry-forward wash sales, basis adjustments, and open positions from prior years into current year.

## Performance Considerations

**Wash sale engine** — For active traders with 50K+ transactions, the iterative wash sale detection could be slow if implemented as N² scanning. Use:
- Sort all transactions by date + symbol
- Build an indexed map: symbol → sorted list of transactions
- For each loss sale, binary search the 61-day window in the symbol's sorted list
- This reduces wash sale detection from O(N²) to O(N log N)

**Lot consumption** — Maintain an in-memory sorted lot queue per symbol per account. Lot selection is O(1) for FIFO/LIFO (head/tail of sorted list) and O(N) for HIFO/Minimum Tax (requires scanning all lots).

**Cross-account scanning** — The wash sale engine scans all accounts. For users with 10+ accounts, pre-build a unified symbol → transactions index across all accounts, then partition results by account for basis adjustment routing.

## Testing Strategy

Each component needs test cases covering:

**Lot engine:** Basic buy/sell FIFO, LIFO, HIFO, specific ID, average cost. Partial lot consumption. Multiple lots consumed by single sale. Stock split adjusting lots. Return of capital reducing basis to zero then excess.

**Wash sale engine:** Same-account basic wash sale. Cross-account (taxable to taxable). IRA permanent disallowance. Partial wash sale (60 of 100 shares). Daisy-chain (wash sale creating new wash sale). Cross-year carry-forward. Short sale wash sale. Options-to-stock (Method 1 and Method 2). User-defined equivalence group. 30-day lookback priority. Convergence on complex chains.

**Section 1256:** Year-end deemed sale. 60/40 split. Qualifying symbol auto-detection. Non-qualifying symbol exclusion.

**Section 475 MTM:** Year-end deemed sale of all positions. Ordinary income classification. Segregated investment exemption. Section 481(a) transition adjustment.

**Section 988:** Ordinary income/loss classification. Opt-out election converting to capital. Currency futures as Section 1256 regardless.

**Corporate actions:** Forward split. Reverse split with fractional cash-in-lieu. Stock-for-stock merger. Cash merger. Mixed merger. Spinoff with allocation. Return of capital exceeding basis.

**Options:** Long call exercise creating stock lot. Covered call assignment. Put exercise. Expiration (long loss, short gain). Section 1256 index option identification.

**Integration:** Full-year scenario with buys, sells, wash sales, dividends, splits, and MTM — verify Form 8949, Schedule D, Form 4797, Form 6781, and Schedule B are all generated correctly.
